## Goal
Expose a first-class object API for opening, inspecting, and typing git objects beyond the current per-type helpers.

## Acceptance Criteria
- [ ] Each port exposes a generic object API for opening objects by OID and inspecting id, type, and size.
- [ ] The API supports peeling tags and otherwise navigating between generic and typed object views.
- [ ] Tests cover loose and packed objects plus tag peeling behavior.
- [ ] The object API composes cleanly with repository, revision, and history workflows instead of duplicating per-type lookup logic.

## Notes
- README parity targets still include `object`, but the current ports mainly expose separate blob/tree/commit/tag helpers.
- This ticket is lower priority than the product-level replacement blockers, but it closes a real parity hole.

## Implementation Plan
- [ ] Define a shared object abstraction for the three ports.
- [ ] Implement object lookup, typing, size inspection, and peel behavior.
- [ ] Reuse existing typed parsers instead of introducing duplicate object decoding logic.
- [ ] Add parity tests for loose, packed, and tag-peeling scenarios.
