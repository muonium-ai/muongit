## Goal
Ship a user-facing CLI that can cover the core git workflows on top of the muongit libraries.

## Acceptance Criteria
- [ ] A blessed `muongit` CLI binary exists with a documented command surface and installation story.
- [ ] The CLI covers the core replacement workflows built by the underlying tickets: init, status, add/remove, commit, branch/switch, reset/restore, log, clone/fetch/push, and diff-related operations.
- [ ] Command output, exit codes, and error handling are documented and validated with cross-platform smoke tests.
- [ ] The CLI never shells out to the stock `git` binary for the workflows it claims to own.

## Notes
- This is the product-level entrypoint ticket. It depends on the underlying feature tickets rather than replacing them.
- Decide early whether the project ships one blessed CLI implementation or one CLI per port with a common contract.

## Implementation Plan
- [ ] Define the CLI contract, supported commands, output conventions, and per-port strategy.
- [ ] Add the executable target(s) and wire them to the underlying library APIs.
- [ ] Add smoke tests that exercise real repository workflows without shelling out to `git`.
- [ ] Document install, usage, and known gaps.
