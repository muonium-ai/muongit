## Goal
Provide high-level staging and commit creation workflows so users can add, remove, stage, and commit changes without assembling raw index entries by hand.

## Acceptance Criteria
- [ ] Each port exposes high-level add, remove, unstage, and commit APIs that operate on repository paths and pathspecs.
- [ ] Commit creation writes the commit object, updates branch refs and reflog state, and uses index/worktree state instead of raw manual object assembly.
- [ ] Tests cover tracked edits, untracked files, deletes, pathspec-based staging, and commit creation on a normal branch.
- [ ] The user-facing workflow no longer requires callers to construct raw index entries by hand.

## Notes
- The current code has low-level object and index helpers, but not the ergonomic repository workflow users expect.
- Keep the API portable across Rust, Swift, and Kotlin instead of baking CLI-only behavior into one port.

## Implementation Plan
- [ ] Design repository-oriented add/remove/unstage/commit APIs on top of existing index and object helpers.
- [ ] Implement path/pathspec-based staging behavior and branch/ref updates during commit.
- [ ] Add reflog updates and error handling for unborn or detached HEAD states.
- [ ] Add parity tests for staged, unstaged, and delete-heavy workflows.
