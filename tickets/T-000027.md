## Goal
Create a benchmark report generator that outputs human-readable timestamped Markdown reports, enabling easy comparison of performance across builds over time.

## Acceptance Criteria
- [ ] Python script `benchmarks/generate-report.py` reads JSONL results and produces Markdown
- [ ] Report filename includes timestamp: `benchmarks/reports/report_YYYYMMDD_HHMMSS.md`
- [ ] Report contains: header with date/time/machine info, 4-way comparison table (libgit2, Rust, Swift, Kotlin), ratio vs libgit2 baseline, performance notes
- [ ] Reports are committed to repo so changes can be tracked across builds
- [ ] `run-all.sh` calls the report generator after benchmarks complete

## Notes
- JSONL results are already saved per-language in `benchmarks/results/{libgit2,rust,swift,kotlin}.jsonl`
- Reports should be diff-friendly so git can show what changed between runs
- Include machine info (platform, CPU, memory) for context

## Implementation Plan
- [ ] Create `benchmarks/generate-report.py` that reads JSONL and outputs Markdown with table, ratios, and timestamps
- [ ] Create `benchmarks/reports/` directory with `.gitkeep`
- [ ] Update `benchmarks/run-all.sh` to call the report generator after all benchmarks complete
- [ ] Run end-to-end and verify report output
