## Goal
Support common working-tree state transitions such as switching branches, checking out paths, resetting state, and restoring files.

## Acceptance Criteria
- [x] Users can switch branches and checkout paths through repository-oriented APIs instead of only low-level index-to-workdir helpers.
- [x] Reset supports the core soft, mixed, and hard semantics for HEAD, index, and worktree state transitions.
- [x] Restore supports selected-path restoration for worktree and index use cases.
- [x] Cross-port tests cover branch switches, detached HEAD, hard reset, path restore, and failure cases involving conflicts or missing paths.

## Notes
- Existing checkout primitives are necessary but not sufficient for the user-facing workflows behind `switch`, `reset`, and `restore`.
- This ticket depends on the branch/refdb surface being available.

## Implementation Plan
- [x] Build branch-switching and detached-HEAD transitions on top of branch/refdb plus checkout primitives.
- [x] Implement reset modes that move HEAD, index, and worktree with correct semantics.
- [x] Implement restore semantics for both path-level and staged-state workflows.
- [x] Add parity tests for common and destructive state transitions.
