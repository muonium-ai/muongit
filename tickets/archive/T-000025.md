## Goal
Set up a GitHub Actions CI/CD pipeline that builds all three ports, runs tests, and creates tagged releases with artifacts.

## Acceptance Criteria
- [ ] CI workflow (`.github/workflows/ci.yml`) runs on push/PR, builds and tests all three ports
- [ ] Release workflow (`.github/workflows/release.yml`) triggers on version tag (`v*`), creates GitHub Release
- [ ] Release reads version from `VERSION` file and validates it matches the tag
- [ ] Release includes build artifacts (Rust binary, Kotlin JAR, Swift library)
- [ ] CI matrix covers macOS (Swift+Rust+Kotlin) and Linux (Rust+Kotlin)

## Notes
- Swift tests require macOS runner (no Linux Foundation support used)
- Kotlin needs JDK 17+ and Gradle wrapper
- Rust needs stable toolchain
- Consider caching dependencies (Gradle, Cargo, SwiftPM) for faster CI

## Implementation Plan
- [ ] Create `.github/workflows/ci.yml` with build+test jobs for all three ports
- [ ] Create `.github/workflows/release.yml` triggered by `v*` tags
- [ ] Add version validation step (tag matches VERSION file)
- [ ] Add artifact build and upload steps for each port
- [ ] Test CI workflow by pushing to a branch
