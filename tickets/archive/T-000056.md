## Goal
Support revision expressions and commit traversal needed for log, history inspection, and range-based workflows.

## Acceptance Criteria
- [x] Each port can resolve common revision expressions such as OIDs, refs, `HEAD~N`, `HEAD^`, and basic two-dot / three-dot ranges.
- [x] Each port exposes history-walking APIs suitable for log-style traversal with stable ordering rules.
- [x] Tests cover revision parsing, merge-base-adjacent range behavior, and commit-walk ordering on fixture repositories.
- [x] The resulting APIs are sufficient to power future `log`, range-based diff, and history-inspection CLI commands.

## Notes
- Users still need revision resolution and history walking even when object access already works.
- Keep the initial scope focused on the most common Git revision expressions before expanding to the full revision language.
