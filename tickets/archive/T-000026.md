## Goal
Build a benchmark harness that compares SHA creation/comparison across all three ports, then incrementally covers other features (OID, tree, commit, index, diff) with comparable metrics.

## Acceptance Criteria
- [ ] Benchmark harness exists for each port (Rust/Swift/Kotlin) with consistent output format
- [ ] SHA-1 and SHA-256 benchmarks: hash 1MB, 10MB data with iterations and median/mean/min reporting
- [ ] OID creation benchmark: 10K-100K object hashes with throughput (ops/sec)
- [ ] Tree serialize/parse benchmark: 1K and 10K entries
- [ ] Commit serialize benchmark: 10K commits
- [ ] Index read/write benchmark: 1K and 10K entries
- [ ] Diff benchmark: 1K and 10K entry tree diffs
- [ ] Output is machine-parseable (JSON or CSV) for cross-language comparison
- [ ] A top-level script or Makefile target runs all benchmarks and produces a comparison report

## Notes
- Start with SHA benchmarks, then gradually add features
- Use warm-up iterations to avoid JVM cold-start skew in Kotlin
- Rust can use `std::time::Instant`, Swift `CFAbsoluteTimeGetCurrent`, Kotlin `System.nanoTime`
- Report format: operation, language, iterations, min_ms, mean_ms, median_ms, throughput_ops_sec
- Existing perf tests are basic assertions; benchmarks should be more rigorous with multiple iterations

## Implementation Plan
- [ ] Create `benchmarks/` directory structure with per-language benchmark runners
- [ ] Phase 1: SHA-1 and SHA-256 benchmarks (hash throughput, creation comparison)
- [ ] Phase 2: OID creation and blob hashing benchmarks
- [ ] Phase 3: Tree, commit, and index serialization benchmarks
- [ ] Phase 4: Diff and delta application benchmarks
- [ ] Create top-level `Makefile` or script to run all benchmarks and aggregate results
- [ ] Output JSON results to `benchmarks/results/` for comparison
