## Goal
Preserve every full benchmark run by writing per-run results and consolidated reports into a timestamped directory instead of overwriting shared outputs.

## Acceptance Criteria
- [x] A full `benchmarks/run-all.sh` execution creates a new timestamped run directory for that invocation instead of overwriting shared JSONL result files.
- [x] The timestamped run directory contains the raw per-implementation outputs for `libgit2`, `rust`, `swift`, and `kotlin`, plus the consolidated comparison artifacts for that run.
- [x] Historical benchmark runs remain intact after multiple executions, so two consecutive full runs produce two separate preserved directories.
- [x] Generated benchmark artifacts no longer live as standalone files at the root of the legacy `benchmarks/results` or `benchmarks/reports` paths.
- [x] Existing benchmark record files are stored under `benchmarks/runs/<timestamp>/...`, and the benchmark tree no longer relies on root-level generated report/result files.

## Notes
- The current benchmark flow overwrites `benchmarks/results/{libgit2,rust,swift,kotlin}.jsonl` on every full run, which loses the raw inputs behind older reports.
- Keep the storage layout simple and diffable. A structure like `benchmarks/runs/YYYYMMDD_HHMMSS/` with both raw outputs and generated reports for that run is sufficient.
- Avoid keeping generated record files such as `latest.md`, per-language JSONL outputs, or timestamped report files directly under `benchmarks/results` or `benchmarks/reports`.
- Generated run directories are ignored via `benchmarks/runs/.gitignore`, so full benchmark runs can accumulate locally without polluting normal git status output.
