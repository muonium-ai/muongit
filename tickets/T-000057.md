## Goal
Provide first-class patch and apply primitives so diff output can be consumed as structured patch data and applied back to repositories or worktrees.

## Acceptance Criteria
- [ ] Each port exposes a structured patch model rather than only formatted diff text.
- [ ] Each port can apply patches back to a worktree and/or index with conflict or rejection reporting.
- [ ] Tests cover add, delete, modify, and context-mismatch cases for patch application.
- [ ] The public APIs are consistent across ports and usable by later CLI workflows without shelling out.
