## Goal
Reduce OID creation and blob hashing from 4-13x slower to within 2x of libgit2.

## Benchmark Data
| Operation | libgit2 | Rust | Swift | Kotlin | Worst gap |
|-----------|---------|------|-------|--------|-----------|
| oid_create_1k | 0.149ms | - | - | 1.914ms | Kotlin 13x |
| oid_create_10k | 1.611ms | 12.9ms | 6.1ms | - | Rust 8x |
| oid_create_100k | 15.8ms | 127.8ms | 60.5ms | - | Rust 8x |
| blob_hash_1k | 0.154ms | - | - | 1.482ms | Kotlin 10x |
| blob_hash_10k | 1.643ms | 12.8ms | 6.4ms | - | Rust 8x |

## Root Cause
OID creation calls SHA-1 hash internally. The bottleneck is the pure SHA implementation (see T-000028). Secondary overhead comes from string formatting for the git object header ("blob <len>\0").

## Acceptance Criteria
- [ ] OID creation within 2x of libgit2 after SHA fix (T-000028)
- [ ] Profile remaining overhead after SHA fix — optimize header construction if needed
- [ ] All OID tests pass

## Implementation Plan

- [ ] After SHA fix, re-benchmark to measure remaining gap
