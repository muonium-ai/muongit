## Goal
Reduce index read/write from 18-5685x slower to within 3x of libgit2.

## Benchmark Data
| Operation | libgit2 | Rust | Swift | Kotlin | Worst gap |
|-----------|---------|------|-------|--------|-----------|
| index_rw_1k | 1.155ms | 2.397ms (2x) | 2.334ms (2x) | 431ms (373x) | Kotlin 373x |
| index_rw_10k | 10.046ms | 196ms (20x) | 181ms (18x) | 57,108ms (5685x) | Kotlin 5685x |

## Root Cause
Index write computes a SHA-1 checksum over the entire index file. This checksum computation dominates the runtime:
- **Kotlin (5685x)**: Pure Kotlin SHA-1 is ~300x slower per byte than OpenSSL, and the 10K index is ~500KB
- **Rust/Swift (18-20x)**: Pure SHA over ~500KB index data. At 1K entries (smaller data), Rust/Swift are within 2x
- **libgit2**: Uses OpenSSL SHA-1 for the checksum — essentially free at these data sizes

## Acceptance Criteria
- [ ] Index R/W 10K within 3x of libgit2 for Rust and Swift (after T-000028 SHA fix)
- [ ] Index R/W 10K within 10x of libgit2 for Kotlin (after T-000028 SHA fix)
- [ ] All index read/write tests pass

## Implementation Plan
- [ ] Primary fix: T-000028 (platform SHA) will resolve most of the gap
- [ ] After SHA fix, re-benchmark to measure remaining overhead
- [ ] If still >3x, optimize index binary serialization (pre-allocate buffers, reduce copies)
- [ ] Kotlin: verify java.security.MessageDigest SHA-1 brings index perf in line
- [ ] Re-run benchmarks and update report
