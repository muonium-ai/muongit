## Goal
Fix Swift OID comparison from 6x slower to within 1.5x of libgit2 (Rust already beats libgit2).

## Benchmark Data
| Operation | libgit2 | Rust | Swift | Kotlin |
|-----------|---------|------|-------|--------|
| oid_cmp_256x16k | 5.875ms | <0.001ms | 34.091ms | 7.529ms |

## Root Cause
Swift OID stores bytes as `[UInt8]` (heap-allocated Array). Comparing two OIDs requires element-wise comparison of Array contents. Rust uses `[u8; 20]` (stack-allocated fixed array) which compiles to a single `memcmp`. Kotlin uses `ByteArray` which also has efficient comparison.

## Acceptance Criteria
- [ ] Swift OID compare benchmark within 1.5x of libgit2 (~8.8ms or less)
- [ ] All existing OID tests pass
- [ ] No API changes to OID public interface

## Implementation Plan
- [ ] Change Swift OID internal storage from `[UInt8]` to a fixed-size tuple or use `withUnsafeBytes` + `memcmp` for Equatable conformance
- [ ] Consider using `Data` instead of `[UInt8]` for storage (contiguous memory)
- [ ] Alternatively, implement custom `==` that uses `withUnsafeBufferPointer` for direct memory comparison
- [ ] Re-run benchmarks and verify improvement

## Notes
- Rust already beats libgit2 at OID compare (<0.001ms vs 5.9ms) — zero-cost abstraction
- Kotlin is within 1.3x — acceptable
- This fix is Swift-only
