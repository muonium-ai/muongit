## Goal
Expose first-class branch and ref database APIs needed for user-facing branch management and repository plumbing.

## Acceptance Criteria
- [ ] Each port exports first-class branch APIs for create, lookup, list, rename, delete, and upstream-tracking operations.
- [ ] Each port exports a refdb-oriented API surface suitable for repository plumbing instead of only loose helper functions.
- [ ] Tests cover loose refs, packed refs, symbolic refs, branch creation from HEAD, detached HEAD behavior, and rename/delete edge cases.
- [ ] The public APIs are consistent enough across ports to support later CLI work without per-port divergence.

## Notes
- README parity targets still mention `branch` and `refdb`, but these are not yet first-class modules in the ports.
- This ticket is foundational for branch-oriented porcelain and remote-tracking workflows.
